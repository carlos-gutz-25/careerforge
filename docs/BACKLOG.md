- **Privacy gate matching semantics** *(parked 2026-07-15, M1-01 kickoff)*: two narrowings shipped to date (added-lines-only, M0-09; lockfile exclusion, M0-10) — each sound in isolation, but the trend is category exclusions for what is really a matching-semantics problem: short real-profile tokens substring-colliding with public strings. **Trigger:** on the NEXT false positive, the fix is word-boundary or distinctiveness-based matching in `scripts/privacy-check.mjs`, not a fourth exclusion. Word-boundary matching was the wrong tool for the actual instance (it was a whole-word match). Probe evidence in the ledger. **ADDENDUM (2026-07-15, M1-02 branch — the queued ratchet review):** three properties of the distinctiveness fix, recorded on second look. (1) **The subtraction is a ratchet, and self-blinding after a miss:** base vocabulary only grows — once a string exists in the base tree the gate permanently stops probing it, so if a real-profile token ever lands on master (through this gate or around it), every future branch republishing it scans clean. Accepted, not fixed: a base-tree occurrence is by definition a pre-existing incident whose remedy is rotation/history-purge (RISKS S-03 procedure), not per-branch detection; the compensating controls that do NOT subtract stay in force (gitleaks pre-commit + CI, the tracked-file guard, and the added-lines scan itself for every token not yet in base). (2) **The M0-10 lockfile exclusion is NOT subsumed by base-vocabulary subtraction** — implementer corrects reviewer here, and the kickoff-review "subsumed" framing is explicitly retired (the falsification loop now runs in every direction): subtraction covers only tokens already in the base tree, while installing a NEW dependency adds registry identifiers absent from base — dropping the exclusion would recreate the exact M0-10 substring-collision class on the next dependency bump. Both mechanisms stay. (3) **`git grep -I`** (script change riding this addendum): without `-I`, binary files in the base tree contribute `Binary file <path> matches` lines to the subtraction corpus, letting file PATHS blind the gate. Gate-modification law satisfied by scratch-repo probes (planted fictional token throughout): a binary named with the token made the pre-change script MISS the token when a branch added it in text (exit 0 — demonstrated self-blinding) while the `-I` script detects it (exit 1); plain planted-token detection 1→1 (kept); clean branch 0→0.
